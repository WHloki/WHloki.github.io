模式匹配算法——KMP的从理解到不懂到再会的历程
发表于 2020-04-07| 更新于 2020-04-17|study
阅读量:

这篇博文基于我之前的数据结构那篇文章，里面的”串”那一章最后剩下的KMP算法没有讲；

今天上课，老师成功地把我会的KMP讲成不会的啦（没说讲的不好）🙃

BF算法

先讲讲BF算法：

暴风(Brute Force)算法是普通的模式匹配算法，BF算法的思想就是将目标串S的第一个字符与模式串T的第一个字

符进行匹配，若相等，则继续比较S的第二个字符和 T的第二个字符；若不相等，则比较S的第二个字符和T的第一

个字符，依次比较下去，直到得出最后的匹配结果。BF算法是一种暴力算法。

看下它的代码实现：

#include <stdio.h>
#include <string.h>

int BF(char *S, char *T)
{   int i = 0, j = 0, k; 
    while (i < strlen(S) && j < strlen(T)) 
    { 
        if (S[i] == T[j]) 
        { 
            i++; 
            j++; 
        }
        else 
        { 
            i = i - j + 1; j = 0; 
        }
    } 
    if (j >= strlen(T)) 
    { 
        k = i - strlen(T) + 1; 
    } 
      else
    {
        k = 0; 
    } 
    return k; 
}

int main() 
{
    int pos;
    char S[100],T[100];
    scanf("%s %s",S,T);
    pos = BF(S,T);
    if(pos) 
    {
        printf("S存在与T相同的子串，位置为第%d个\n",pos);
    }
    else
    {
        printf("S不存在与T相同的子串\n");
    }
    return 0;
}

这种算法也是可以找到匹配的位置，但是效率不高，时间复杂度为O(M*N);

KMP算法

既然BF算法效率不高，那就寻找一种简便的方法来实现相同的功能——KMP；

KMP算法：D.E.Knuth、J.H.Morris、V.R.Pratt发表的一个模式匹配算法，可以大大避免重复遍历（减少回

溯）的情况，称之为克努特—莫里斯—普拉特算法，简称为KMP算法；

直接上代码，这里没法解释

获取next数组：

void Get_Next(char *T,int *next)
{   
    int i,j;
    next[1] = 0;
    i = 1;
    j = 0;
    while(i<strlen(T))
    {
        if(j==0||T[i]==T[j])
        {
            i++;
            j++;
            next[i] = j;
        }
        else
        {
            j = next[j];
        }
    }
}

注意！！！！

这里数组下标是从1开始的，但是数组默认从0开始，所以在调用这个函数的时候，不能直接：

// scanf(“%s”,S); Get_Next(S,next);

如果这样写，100%错的（自闭一晚上才看出来，可见博主是多么菜）😭😭

然后，就…没有然后了，555555

算了，还是改进一下吧，把数组从0开始赋值：

void Get_Next(char *T,int *next)
{   
    int i,j;
    next[0] = -1;
    i = 0;
    j = -1;
    while(i<strlen(T)-1)
    {
        if(j==-1||T[i]==T[j])
        {
            i++;
            j++;
            next[i] = j;
        }
        else
        {
            j = next[j];
        }
    }
}

这样写就可以直接输入串了，不用考虑下标问题了……

KMP

无话可说，代码很好懂😀

int KMP(char *S,char *T,int pos)
{
    int i = pos;
    int j = i;
    int next[100];
    Get_Next(T,next);
    while(i<=strlen(S)&&j<=strlen(T))
    {
        if(j==0||S[i]==T[i])
        {
            i++;
            j++;
        }
        else
        {
            j = next[j];
        }
    }
    if(j>strlen(T))
    {
        return i-strlen(T);
    }
    else
    {
        return 0;
    }
}

KMP算法的时间复杂度为O(M+N);

对比BF算法的时间复杂度，显然，KMP好一些；
