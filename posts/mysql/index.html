2、也可以直接打开MySQL Command Line Client，输入密码就可以使用

MySQL的层次结构

DBMS：DataBase Management System 数据库管理系统
DB：DataBase 数据库
Table：数据表对应现实中的表格
Record：数据表中的一行
Fields：数据表中的一列

数据库的操作

1、查看当前有多少数据库

语法：show databases;

2、创建数据库

语法：create database 数据库名 库选项;

注：如果数据库名与关键字重名，则必须使用反引号（``）将数据库名括起来，但是一般不建议重名，便于区分与理解。

库选项：

● charset：用于设置存储字符集

● collate：用于设置校验集

例如：

3、查看数据库的创建语句

语法：show create database 数据库名;

4、删除数据库

语法：drop database 数据库名;

5、选择数据库

语法：use 数据库名;

注： \s; 可以查看数据库的信息，其中包含当前所在的数据库。

数据表的操作（use 数据库）

1、设置客户端的通讯编码

语法：set names 客户端的字符集;

2、查看数据表（查看数据库中有多少个数据表存在）

语法：show tables；

3、创建数据表

语法：create table 表名(

字段名 列类型 列属性

字段名 列类型 列属性

)表选项;

表选项：

● charset：设置存储字符集

● collate：设置校验集

● engine：存储引擎

4、查看表结构（内容）

语法：desc 表名;

5、查看表的创建语句

语法：show create table 表名;

或者：show create table 表名\G;

6、删除表

语法drop table 表名;

数据的基本操作

1、增加数据

语法：insert into 表名(字段列表) value(值列表);

字段列表中字段名不需要加引号，

值列表中的字符型的值必须加引号，

字段列表中字段的个数要与值列表中值的个数一致，顺序也要一致，

字段列表也可以省略，如果省略那表示全部字段插入数据，值列表中的值的顺序与表中的字段的顺序一致。

例如：

2、查看数据

语法：select * 字段列表 from 表名;

● 查看所有字段：select * from 表名

● 查看部分字段：select 字段名1，字段名2… from 表名

★ where子句：select * from 表名 where 子句

子句由：字段名 运算符组成

运算符有：

① >、<、=、<>、>=、<=

② and、or 逻辑与、逻辑或

③ between m and n 字段值介于m和n之间

④ in(值列表) 字段的值等于值列表中的某一个

⑤ is null、is not null 判断值是否为null的操作

3、修改数据

语法：update 表名 set 字段1=值1，字段2=值2 where 子句;

修改age为18的数据为21；

4、删除数据

语法：delete from 表名 where子句;

数据类型

小数型

小数型分为浮点与定点

浮点型又分为float和double

例如：

m表示总位数，d表示小数位数，m-d表示整数位数

对于浮点型会出现精度丢失问题：

float：精确5-6位，多余的位会用0代替

double：精确14-15位，多余的位会用0代替

字符型

定长

语法：char(len)

len 表示字符的个数，最大是255

变长

语法：varchar(max)

max 表示字符的个数，理论上是最大65535

文本类型

text是一个系列，包含：

tinytext、text、mediumtext、longtext

区别在于能存储的字符的多少不同

blob类型

blob是一个系列，包含：

tinyblob、blob、mediumblob、longblob

区别在于能存储的数据的多少不同

blob可以存储二进制数据

enum：用于规范数据，只允许用户从多个预定的数据中，选取一个

语法：enum(值列表)

日期时间型

year：2或4 年份 2表示2位的年份，4表示4位的年份

date： 日期 格式：’ 2020-8-19’

time： 时间 格式：'18:26:00’

datetime： 日期时间 格式：'2020-8-19 18:26:00’

timestamp： 时间戳 主要是记录数据表中最后一次被修改的时间

字符集与校验集

字符集

查看MySQL支持的字符集：

语法：show charset;

校验集

校验集就是某一种字符集中的比较规则。

语法：show collation

校验集以3中方式结尾：

_cs：CaseSensitive 大小写敏感（区分大小写）

_ci：CaseInsensitive 大小写不敏感（不区分大小写）

_bin：二进制 区分大小写

修改表结构

总语法：alter table 表名 add/drop/modify/change/rename

操作 作用
add … 添加字段
add primary 添加主键
drop… 删除字段
drop primary key 删除主键
modify 修改字段类型
change 修改字段名
rename 表重命名

数据源

数据源就是数据的来源

数据源可以分为单表数据源、多表数据源、子查询数据源：

单表数据源：数据来源只有一个表

多表数据源：数据来源于多个表

子查询数据源：from后数据的来源是另一个select语句

单表数据源

单表数据源就是普通的查询方法：select * from 表名;

多表数据源

语法：select * from 表A,表B;

说明：多表数据源产生的结果：笛卡尔积：两个表的字段数相加，记录条数相乘

子查询数据源

表中的数据是存储在硬盘中的一个文件中。

表别名

在一条select语句中引用另一个表，也可以为这个表起一个别名，主要应用在连接查询中

select 五子句

select 五子句：[where子句] [group by子句] [having子句] [order by子句] [limit子句]

这些子句可以随意组合使用，但是顺序必须按照这个语法顺序书写，否则会报错
